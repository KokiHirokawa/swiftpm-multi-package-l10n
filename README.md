# swiftpm-multi-package-l10n

This is a sample project demonstrating localization (l10n) in a SwiftPM multi-package setup using `xcstringstool`.
Localized resources are centralized in a single package and shared across the project.

## Development Environment

- Xcode 26.1.1 (17B100)
- Swift 6.2.1

## Package Structure

```
Packages/
├── Root/            # Root package
├── FeatureA/        # Feature A
├── FeatureB/        # Feature B
└── UIResources/     # Centralized localization resources
```

### UIResources

The package that aggregates all localized strings.

- `Localizable.xcstrings` - Multilingual string definitions
- `GeneratedStringSymbols_Localizable.swift` - Type-safe symbols auto-generated by `xcstringstool`

## How to Localize

```swift
import UIResources

Text(.fATitle)
```

## How to Generate Localization Symbols

After editing `Localizable.xcstrings`, regenerate type-safe symbols with:

```bash
make generate-symbols
```

This command uses `xcstringstool generate-symbols` to auto-generate `LocalizedStringResource` extensions.
